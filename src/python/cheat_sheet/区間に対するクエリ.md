# 区間クエリデータ構造 対応表（計算量付き）

区間クエリを解くための主要なデータ構造と、それぞれの得意・不得意をまとめた比較表です。

## 1. 機能比較表

| クエリ種別 | MergeSortTree | Wavelet Matrix | 永続セグメント木 | セグメント木 / BIT | Mo's Algorithm |
| :--- | :---: | :---: | :---: | :---: | :---: |
| **区間和 / 最大・最小** | △ $O(\log^2 N)$ (注1) | × | ○ $O(\log A)$ | **◎ $O(\log N)$** | ○ $O(1)$ (伸縮時) |
| **区間内の $x$ の個数** | ◎ $O(\log^2 N)$ | **◎ $O(\log A)$** | ○ $O(\log A)$ | △ $O(\log N)$ (注2) | ◎ $O(1)$ (伸縮時) |
| **区間内 $k$ 番目の値** | ○ $O(\log^2 N \cdot \log A)$ | **◎ $O(\log A)$** | **◎ $O(\log A)$** | × | ○ $O(\sqrt{N})$ |

## 計算量に関する詳細解説

### 1. MergeSortTree の $O(\log^2 N)$
セグメント木の各ノード（高さ $\log N$）において、`bisect`（二分探索 $\log N$）を行うため、この計算量になります。
- $k$-th smallest を求める場合は、さらに外側で「値」に対する二分探索を行うため、$O(\log^2 N \cdot \log A)$ と重くなります。

### 2. Wavelet Matrix / 永続セグメント木の $O(\log A)$
これらは数値の「ビット（桁数）」に沿って探索するため、配列サイズ $N$ ではなく、値の範囲 $A$ の対数に依存します。
- 座標圧縮を事前に行うことで、この $A$ を $N$ に置き換えることが可能です（$O(\log N)$）。



### 3. Mo's Algorithm の特殊な計算量
Mo's Algorithm は「構造」ではなく「クエリの解き方（並び替え）」の工夫です。
- 全体の計算量は $O(N\sqrt{Q})$ となり、一点あたりの伸縮はならし $O(1)$ で計算できる場合に非常に強力です。
- 他のデータ構造が苦手な「種類数」や「最頻値」で真価を発揮します。



---

## 補足
- **BIT（Fenwick Tree）** はセグメント木より定数倍が非常に速く、メモリも半分で済むため、区間和や点更新がメインの場合は第一選択となります。
- **Wavelet Matrix** は、メモリ制限が厳しい（256MB以下など）かつ $N, Q \ge 2 \times 10^5$ のような問題で、MergeSortTree に代わる強力な選択肢になります。

## 2. 各データ構造の解説

### MergeSortTree
- **得意**: 「$x$ 以下の値の個数や総和」など。
- **特徴**: セグメント木の各ノードにソート済み配列を持たせる。実装が比較的シンプルで、総和計算との相性が良い。
- **弱点**: 更新がほぼ不可能。メモリ消費が $O(N \log N)$ とやや大きい。

### Wavelet Matrix
- **得意**: **「$k$ 番目の値」「範囲内の個数」**。
- **特徴**: 値のビット表現を利用して構築される。空間計算量が $O(N)$（ビットベクトル）で済むのが最大のメリット。
- **弱点**: 「総和」を求めるにはノードごとに累積和の配列が必要になり、メモリメリットが薄れる。



### 永続セグメント木 (Persistent Segment Tree)
- **得意**: **「過去の状態のクエリ」** および 「k-th smallest」。
- **特徴**: 各要素の更新履歴を保存し、任意の時点のセグメント木として扱える。
- **弱点**: メモリ消費が激しい。

### セグメント木 / BIT (Fenwick Tree)
- **得意**: **「更新ありの区間和・最大値」**。
- **特徴**: 競技プログラミングの基本。代数的構造（モノイド）であれば汎用的に使える。
- **弱点**: 「値の順序（k-thなど）」に関するクエリには単体では対応できない。

### Mo's Algorithm (クエリ先読み・平方分割)
- **得意**: **「種類数」「最頻値」** など、他の構造では解きにくい複雑なクエリ。
- **特徴**: クエリを並び替えて、区間の端を伸縮させて解く。
- **弱点**: **オフライン（全クエリが事前に判明している）**である必要がある。

---

## 3. 補足事項
- **(注1)**: 各ノードに累積和を持てば可能だが、計算量が $O(\log^2 N)$ になりセグメント木より遅い。
- **(注2)**: 値を座標圧縮してセグメント木に乗せれば可能だが、基本的には MST 等の方が適している。
- **(注3)**: 個数は得意だが、総和を出すには累積和の配列が必要になり、メモリを $O(N \log A)$ 消費する。
- **(注4)**: クエリを右端でソートして「前回の出現位置」を BIT で管理するテクニック（Offline BIT）が必要。
- **(注5)**: 動的 Wavelet Matrix も存在するが、実装が極めて複雑。
- **(注6)**: 更新あり Mo's (3D Mo's) もあるが、計算量が重くなる。
- **(注7)**: 構築は不要だが、クエリの並び替え $O(Q \log Q)$ と伸縮 $O(N \sqrt{Q})$ がかかる。
